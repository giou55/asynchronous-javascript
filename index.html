Asynchronous JavaScript

Callbacks

Callback is a function that passed as an argument to another function and will be called sometime in the future.
Callback does not execute immediately.
This is the callback hell:
      getProfile((profile) => {
        getUser(country, (user) => {
          getName(user, (name) => {
            ....
          })
        })
      })

Promises

fetch API is the modern equivalent of the XMLHttpRequest.
fetch API uses Promises, which means that returns a Promise object
then() contains a callback function and returns another Promise.
      fetch("https://jsonplaceholder.typicode.com/users")
        .then(response => {
          return response.json();
        })
        .then(jsonData => {
          let users = jsonData;
          console.log(users);
        })
        .catch(error => {
          console.log(error.message);
        });
    
      getProfile()
        .then(getUser)
        .then(getName)
        .then(() => console.log('done'))
        .catch((error) => console.log(error))

Async/Await

Async/Await makes Promises more readable.
The function below became aynchronous with keyword async, and it will return a Promise than a value: 
      async function print(){
        return "Hello";
      }
We can use then() to consume the Promise: 
      print().then(response => console.log(response));
